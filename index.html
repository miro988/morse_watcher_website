App Introduction

Three moments that explain how Morse Watcher works.

Choose the light source

The app starts with a direct tap-to-target flow so detection begins from the exact flashing region you care about.

Calibrate the signal

Region-of-interest tuning and brightness controls help lock onto the source before decoding begins.

Read the output

Live Morse symbols, decoded text, and timing diagnostics make it easier to inspect real-world signal behavior.

Capabilities

A small site that still feels intentional.

Real app visuals

The site now introduces the actual product flow with app screenshots instead of relying only on abstract mock panels.

Static by default

No build step is required. Clone, edit, and publish directly from GitHub without adding tooling before the project needs it.

Pages-ready deployment

The repository includes a GitHub Actions workflow that deploys the site to GitHub Pages from the `main` branch.

Operator notes

signal.locked = true
sample.window = 180ms
decoded.output = "MW"
deploy.target = "github-pages"

Deploy

Publish from this repository with minimal friction.

  1. Edit the content in `index.html`, `styles.css`, and `script.js`.
  2. Push changes to `main`.
  3. In GitHub, set Pages to use GitHub Actions if it is not already selected.
  4. The workflow in `.github/workflows/deploy.yml` publishes the site.
